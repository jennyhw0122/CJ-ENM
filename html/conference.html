대규모 IT 컨퍼런스 UX 설계: SYNXCON 2025

01. 프로젝트 개요

프로젝트명SYNXCON 2025 웹 플랫폼 UX 설계 및 구축
기간2025.02.24 – 2025.04.02 (총 6주)
참여 인원기획 1, 디자이너 2, 프론트엔드 2, 백엔드 4 (총 9명)
담당 역할 UX 전략 수립 및 사용자 플로우 정의
디자인 시스템 설계 및 컴포넌트 개발
프로토타입 제작 및 개발 커뮤니케이션
UT 및 AI 기반 UX 분석 주도

‘기록의 단절’과 ‘네트워킹 부족’이라는 구조적 UX 문제를 해결하기 위한 프로젝트

02. 문제 정의

참가자 27명 중 39.5%가 “행사 인사이트 기록이 번거롭다”고 응답했고,
90명 중 14.3%는 “네트워킹 기회가 부족하다”고 답함

참가자는 세션 내용을 별도 앱에 기록 → 플랫폼과 단절 → 공유 및 네트워킹 흐름 단절

결론: “기록·공유·연결이 유기적으로 이어지는 UX 구조로 재설계 필요”
인사이트 노트 기능 중심으로 디지털 컨퍼런스 경험 확장

03. UX 전략 및 설계 원칙

  • 세션 인사이트 기록 → 공유 → 교류까지 이어지는 핵심 흐름 중심 UX
  • MVP 우선 설계 및 확장 가능성 고려

디자인 시스템

  • Foundation: 색상, 그리드, 타이포 시스템 정의
  • Components: Slot, Tab, Card 등 반응형 대응 컴포넌트 설계
  • UI 일관성: 세션, 노트, 마이페이지 동일한 구조 및 인터랙션
  • 재사용성 강화: 공통 UI 컴포넌트화

04. UX 플로우 및 정보구조 설계

핵심 플로우

세션 탐색
세션 신청
인사이트 노트 작성
마이페이지 공유

정보구조 (IA)

  • 홈: 세션 안내 + 인사이트 노트 강조
  • 세션: 키워드 필터 + 카드형 요약
  • 노트: 실시간 저장 + 공유 기능
  • 마이페이지: 신청내역 및 작성 콘텐츠 확인

05. UX 개선 흐름

기간내용
2/24 ~ 2/28기획 및 와이어프레임, 개발환경 세팅
3/2 ~ 3/10간편 로그인, 세션 리스트 구현
3/11 ~ 3/18인사이트 노트 작성 기능, QR 인증, UT 설계
3/19 ~ 3/26마이페이지 구성 및 UT 피드백 반영
3/27 ~ 3/31QA 및 배포 준비

→ 세션 신청 → 기록 → 네트워킹까지의 여정을 구조화

06. 디자인 결과물

  • 세션 카드 UI / 추천 태그
  • 인사이트 노트 템플릿
  • 마이페이지 개인화 구조

07. AI 기반 정량 UX 분석

인터뷰 어려움을 AI 기반 페르소나로 대체 → 10명 페르소나 설계 및 시나리오 UX 인터뷰 진행

UX 클러스터응답 수도출 인사이트
세션 정보 부족7건KPI 기반 카드 리디자인
로그인 피로도6건반복 입력 제거, 안내 강화
노트 작성 장벽6건템플릿 기반 구조화 필요
저장/공유 흐름 불안5건시각적 피드백 도입
항목정량 지표수치
노트 공유 의향업무 공유 가능성83%

로그인 피로도평균 만족도2.8 / 5

AI 기반 분석은 사용자 불편을 정량화하여 설계의 당위성과 우선순위를 명확히 해주는 도구로 작동

08. 협업 및 갈등 해결 전략

기획·디자인·개발 간 중심 기능 유지 vs. 일정 조율 갈등 발생

인사이트 노트 기능을 단순히 줄이는 대신,
구현 난이도 높은 기능은 후순위로 조정하며 핵심 흐름은 유지
  • 우선순위 도출 기준 공유
  • 유저 여정 기반 핵심 UX 흐름 보존
  • 단순 MVP가 아닌 설계 의도 중심 구조 완성
