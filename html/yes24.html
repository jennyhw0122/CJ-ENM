YES24 디자인 시스템 구축 및 개선 보고서

프로젝트 개요

YES24 디자인 시스템 구축 프로젝트는 사용자 경험 개선을 중심으로 브랜드 일관성, 확장성, 유지보수 효율성을 극대화하기 위해 Foundation, Modes & Variables, Components의 세 가지 핵심 구조를 중심으로 설계되었습니다.

  • 진행 | 프로덕트 디자이너 (5인)
  • 프로젝트 기간 | 2025. 01. 21 - 2025. 02. 21 (4주간)
  • 기여도 | 기획 (UX 전략 수립) 35%, 디자인 (UI 디자인 + 디자인 시스템 구축 + 컴포넌트 제작 + 가이드라인) 65%

문제 정의

기존 YES24 서비스 화면 분석을 통해 다음과 같은 문제점을 발견했습니다:

  • 기획자의 의도
    • 각 서비스(도서, 티켓)의 요구사항이 달라 개별 최적화 중심의 UI를 운영함으로써 UI 일관성이 무너짐
    • 통합된 디자인 원칙과 표준화된 컴포넌트 부재로 서비스 간 불일치 심화
  • 디자이너의 의도
    • 기존 UI 유지가 사용자 혼란 최소화와 운영 안정성 확보에 유리하다고 판단
    • 개별 서비스 최적화를 우선 적용하여 일관성 관리의 어려움 발생
  • 개발자의 문제점
    • 서비스마다 별도의 UI 구조로 인해 코드 중복과 유지보수 난이도 증가
    • 공통 컴포넌트 부재로 신규 기능 추가 및 서비스 확장 시 개발 효율성 저하

※ 시각 자료: 기존 화면 UI 문제점 시각화 이미지 첨부 예정 (컬러, 타이포그래피, Spacing 중심)

사용자 여정 중심 구성

YES24 사용자들이 경험하는 탐색 → 선택 → 구매 단계별로 디자인 시스템의 개선 효과를 제시합니다.

  • 탐색 단계: 컬러 시스템 개선으로 정보 탐색 직관성 향상 → [Before & After 컬러 시스템 화면 이미지 삽입]
  • 선택 단계: 타이포그래피 및 간격(Spacing) 표준화로 선택 과정 명료화 → [Before & After 타이포그래피 비교 이미지 삽입]
  • 구매 단계: 공통 컴포넌트 적용으로 구매 과정 일관성 강화 → [도서와 티켓 동일 컴포넌트 적용 예시 이미지 삽입]

Before vs After 비교 중심 구성

  • 실제 화면을 개선 전후로 나란히 제시하여 시각적으로 명확한 개선 효과를 제공합니다.
  • 컬러, 타이포그래피, Spacing 등의 구체적인 변화와 그로 인한 사용자 경험 개선 효과를 강조합니다. → [슬롯 컴포넌트 개선 전후 적용 예시 삽입]

문제-해결 구조화 매트릭스 구성

문제 중요도 (UX 영향력) 해결 난이도 해결방안
컬러 HEX코드 직접사용 높음 중간 Primitive → Theme → Semantic 구조 적용
타이포그래피 일관성 부족 중간 낮음 타이포그래피 스타일 명확화
컴포넌트 부재 높음 높음 Atomic Design & Slot 시스템 도입

UX/UI 이슈 및 솔루션 중심 구성

  • 타이포그래피 이슈: 스타일 중복과 위계 불명확 → 기준 수립으로 일관성 개선 → [타이포그래피 기준 정의표 및 적용 이미지 삽입]
  • Spacing 이슈: 비일관 단위 사용 → 2px, 4px, 8px 기준 체계로 통합 → [Spacing 단위 적용 기준표 이미지 삽입]

검증 및 프로토타입 제작 과정

YES24 도서 및 티켓 서비스에 디자인 시스템을 실제 적용한 프로토타입 제작 및 사용자 시연 진행
다양한 디바이스 환경에서 UI 시연을 통해 브랜드 간 UX 일관성 검증 → [프로토타입 시연 영상 링크 또는 썸네일 이미지 첨부]

프로젝트 기대 효과

  • 브랜드 간 UI 일관성 확보로 사용자 경험 개선
  • Modes & Variables 시스템으로 서비스 확장성과 유지보수 용이성 확보
  • 컴포넌트 재사용 극대화로 유지보수 비용 절감 및 효율성 향상

→ 최종적으로, 디자인 시스템은 사용자의 학습 비용을 줄이고 브랜드 충성도를 높이는 UX 기반 구조로 기능함.
