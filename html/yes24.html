YES24 디자인 시스템 구축 및 개선

통합된 기준 없이 개별 최적화만 반복된 결과,
브랜드 간 UI 불일치와 유지보수 비효율이 누적되었습니다.

프로젝트 개요

진행프로덕트 디자이너 (5인)
프로젝트 기간2025.01.21 ~ 2025.02.21 (4주)
기여도기획 35% · 디자인 65%

Foundation · Modes & Variables · Components 중심의 시스템 설계

문제 정의

  • 기획자: 각 서비스의 UI 최적화 → UI 일관성 붕괴
  • 디자이너: 기존 UI 유지로 사용자 혼란 최소화 → 일관성 관리 어려움
  • 개발자: 코드 중복 · 유지보수 난이도 증가 → 효율성 저하
    • 시각 자료: 컬러, 타이포그래피, Spacing 문제 시각화 이미지 예정

    사용자 여정 중심 개선

    • 탐색 단계: 컬러 시스템 개선 → 정보 탐색 직관성 향상
    • 선택 단계: 타이포그래피 · Spacing 표준화 → 선택 과정 명료화
    • 구매 단계: 공통 컴포넌트 적용 → 구매 흐름 일관성 강화

    문제-해결 구조화 매트릭스

    문제중요도해결 난이도해결 방안
    컬러 HEX코드 직접 사용 높음 중간 Primitive → Theme → Semantic 구조
    타이포그래피 일관성 부족 중간 낮음 스타일 명확화
    컴포넌트 부재 높음 높음 Atomic Design · Slot 시스템 도입

    UX/UI 이슈 및 솔루션

    • 타이포그래피 이슈: 위계 불명확 → 기준 수립으로 일관성 개선
    • Spacing 이슈: 비일관 단위 사용 → 2px, 4px, 8px 기준 체계로 통합
    타이포그래피 / Spacing 기준표 시각 자료 예정

    검증 및 프로토타입

    • 도서 · 티켓 서비스 프로토타입 제작 및 시연
    • 다양한 디바이스에서 UX 일관성 검증
    프로토타입 시연 영상 링크 / 이미지 예정

    실무 관점 효과

    역할개선 포인트정량/정성 효과
    기획자 컴포넌트 단위 정의로 전달 명확 문서 작성 시간 30%↓, 피드백 3→1
    디자이너 재사용 설계, Variables 활용 티켓에 90% 적용, 컬러 변경 1시간→15분
    개발자 props 기반 구현, QA 피드백 감소 UI 구현 시간 40%↓, 피드백 5→2건

    프로젝트 기대 효과

    • UI 일관성 확보 → 사용자 경험 개선
    • 변수 기반 시스템 → 유지보수 & 확장성 향상
    • 컴포넌트 재사용 → 협업 효율, 개발 시간 절감

    디자인 시스템은 사용자의 학습 비용을 줄이고 브랜드 충성도를 높이는 UX 기반 구조입니다.
