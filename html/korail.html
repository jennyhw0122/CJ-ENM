문제 정의 | 사용자의 ‘선택 불확실성’이 예약을 방해했다

여행을 계획하는 순간은 설레야 하지만, 사용자의 35% 는 예약 도중 떠났다.

코레일톡 기획자의 의도는 ‘지역 기반 기차 여행 활성화’였고, 명확한 예약 플로우를 제공하고자 했다.

하지만 실제 사용자의 행동은 이렇게 흐르고 있었다:

“선택한 정보들이 초기화되어 다시 처음부터 입력해야 해서 피곤했다.”
“뒤로 가면 처음으로 돌아가서 짜증 난다.”

이탈의 원인은 단계 수가 아니라, 선택 이후 흐름이 끊긴 구조에 있었다.

사용자가 원하는 정보를 선택한 후, 다시 사라지는 경험은 감정 흐름을 단절시키고, 서비스에 대한 신뢰도에도 부정적인 영향을 끼쳤습니다.

UX 가설 | 사용자가 고른 정보가 이어질수록, 예약은 끊기지 않는다

출발역과 인원을 한 번에 선택하고, 선택한 정보가 흐름 속에서 유지된다면, 사용자는 반복 선택 없이 예약을 마무리할 수 있을 것이다.

디자이너 관점

선택값 유지 → 심리적 피로 해소 → 반복 입력 제거까지 연결되는 구조 설계 흐름을 스스로 설계하고 실험함. 기능 추가가 아닌 플로우 구조 리디자인이 핵심.

개발자 관점

선택값 유지 로직은 별도 기능 추가 없이 기존 상태 관리 구조를 활용하여 효율적 구현. 유지보수 관점에서도 일관된 UX 제공에 용이.
