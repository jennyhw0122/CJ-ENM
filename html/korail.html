뒤로 가기 3.07회 → 2.00회

사용자 흐름을 재설계해 예약 완료율을 7.18%p 개선한 구조 중심 UX 사례

프로젝트 개요

역할프로덕트 디자이너 (UX 전략 · 조사 · 구조 설계 · UI)
프로젝트 기간2024.12.16 ~ 2025.01.17 (4주간)
팀 구성디자이너 4명· 개발자 1명
기여도UX 전략/조사 40%, UI 설계 50%, 디자인 QA/협업 10%
도구GA4, Figma, Notion, Google 설문, 웹 크롤링

문제 정의 | 사용자의 ‘선택 불확실성’이 예약을 방해했다

여행을 계획하는 순간은 설레야 하지만, 사용자의 35%는 예약 도중 떠났다.

기획 의도는 ‘지역 기반 기차 여행 활성화’였고, 명확하고 빠른 예약 플로우를 제공하고자 했다.

하지만 실제 사용자의 행동은 이렇게 흐르고 있었다:

“선택한 정보들이 초기화되어 다시 처음부터 입력해야 해서 피곤했다.”
“뒤로 가면 처음으로 돌아가서 짜증 난다.”

이탈의 원인은 단계 수가 아니라, 선택 이후 흐름이 끊긴 구조에 있었다.

사용자는 출발역과 인원 등 선택값을 입력했지만, 수정이 필요할 경우 ‘뒤로 가기’ 외에는 방법이 없었고, 이로 인해 선택값이 초기화되며 처음부터 다시 입력해야 했다.

감정 흐름 단절 포인트

이 반복 구조는 선택 변경 불가능 → 정보 상실 → 심리적 피로 → 예약 중단이라는 구조적 UX 이탈 경로를 만들었다.

2. 사용자 조사 | 행동 + 감정 데이터를 통해 흐름을 추적했다

조사 방식

  • 앱 리뷰 크롤링 129건
  • Google 설문 + 미션형 인터뷰 30명

주요 인사이트 (형식: 도넛 차트)

  1. 리뷰 키워드 분석
    예약/UX 문제 비중: 43.1%
    주요 항목: 복잡한 예약 구조: 43.6%
  2. 정량 설문 결과(형식: 점수카드)
    예약 화면 직관성: 2.47점
    예약 단계 과다: 2.53점
    정보 접근성: 2.50점

→ 특히 정보 접근성과 직관성이 낮았던 사용자일수록, 재사용 의향도 낮았음.

→ ‘한 번 써봤는데 복잡해서 다시 쓰고 싶지 않다’는 흐름이 드러남.

3. UX 가설 | 선택값 유지와 시각 요약 구조는, 사용자의 반복 입력 피로를 줄이고 예약 이탈을 방지할 수 있다.

만약 사용자가 출발/도착역과 상세 인원 정보를 한 흐름에서 입력하고, 선택값이 유지되는 구조가 제공된다면, 선택 이후 반복 입력으로 인한 피로가 줄어들고 예약 완료율은 높아질 것이다.

4. Before/After 구조 변화

항목기존 UI개선 UI
인원 선택 위치상품 선택 이후출발역과 통합 배치
선택값 유지초기화됨유지됨

→ 탐색보다 확신을 먼저 제공하는 구조로, 불확실성에 의한 반복 입력을 줄였다.

5. 플로우 변경

기존 플로우 (11단계)
메인화면 → 지역별 여행 → 상품목록 → 상품 상세 → 날짜 선택 → 인원/열차시간 → 패키지 선택 → 출/도착역 선택 → 상세 인원 선택 → 약관 동의 → 예약 정보

개선 후 플로우 (9단계)
메인화면 → 지역별 여행 → 상품목록 → 상품 상세 → 날짜 선택 → 출발·도착역 + 인원 선택 → 상품/패키지 선택 → 약관 동의 → 예약 완료

→ 단계 수가 줄어든 것이 아니라, 감정 흐름이 끊기지 않도록 재배열한 것

6. 실험 결과 | 구조 설계로 행동이 바뀌었다

7. 전략적 확장 가능성

적용 플랫폼기대 효과
티켓팅 플랫폼좌석·일자·인원 선택 통합 → 흐름 최적화
숙박 예약 플랫폼옵션 선택 단계를 구조화하여 이탈 방지
행사 등록 시스템정보 확인 및 수정 구조 → 반복 입력 최소화

→ 핵심은 사용자 불안 요소를 미리 감지하고, 수정 가능성과 정보 유지로 감정을 안정화시키는 구조 설계임

8. 구조 설계의 실무적 가치 요약

“사용자의 선택이 확신으로 바뀌는 그 순간을 설계한 구조 중심 UX 개선 사례입니다.”

뒤로 가기 평균 3.07회 → 2.00회, 예약 완료율 +7.18%p 상승이라는 정량적 변화로 설계 효과를 입증했습니다.

실무적 인사이트로 확장

관점적용 포인트
기획자 관점단계를 줄이는 것이 아니라, 정보의 등장 순서와 확인 구조를 재배열하여 감정 흐름을 설계.
디자이너 관점선택값 유지 → 심리적 피로 해소 → 반복 입력 제거까지 연결되는 구조 설계의 흐름을 스스로 설계하고 실험함. 기능 추가가 아닌 기존 플로우의 구조 리디자인이었기에 디자인 역량의 핵심.
개발자 관점선택값 유지 로직은 별도 기능 추가가 아니라 기존 상태 관리 구조를 활용해 효율적으로 구현 가능. 유지보수 측면에서도 일관된 UX 제공이 용이.
