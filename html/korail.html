프로젝트 개요

도구GA4, Figma, Notion, Google 설문, 웹 크롤링

여행을 계획하는 순간은 설레야 한다.

그런데, 사용자의 35%는 예약 도중 떠날까요?

기획 의도는 지역 기반 기차 여행 활성화를 위해 사용자에게 정확한 예약 흐름을 제공하는 것이었으나,

  • 설문조사 (30명)에서 ‘예약 화면 직관성’ 2.47점, ‘단계 과다’ 2.53점으로 낮은 만족도 확인
  • 인터뷰와 UT에서는 “뒤로 가면 처음부터 다시 시작돼서 짜증”이라는 피드백이 많았다.
  • 앱 리뷰에서도 “뒤로가기가 작동하지 않음”, “예약 구조가 너무 복잡하다”는 반복된 언급
사용자가 인원 선택값을 변경하고 싶어도 해당 플로우에서 변경할 방법이 없어, 반복적으로 ‘뒤로 가기’를 누르는 흐름이 생김.
선택값은 모달 형태로 노출되지만, 수정하려면 뒤로 가기를 눌러야 하며 초기화되어 다시 처음부터 입력해야 했음.

‘선택 변경 불가능한 정보 → 반복 선택 → 피로 → 예약 포기’의 구조적 문제가 UX 이탈의 근본 원인임
